# 系统接口

## 查询配置热加载状态

`GET /internal/system/config-reload/status`

该接口为内网运维路由，不允许公网访问；请求需要登录态和运维令牌，用于查看当前配置热加载监听和最近一次重载结果。

### 请求头

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| Authorization | string | 是 |  | `Bearer <token>` |
| X-Ops-Token | string | 是 |  | 配置热加载运维令牌，需匹配 `ops.config_reload_token` |

### 安全字段

| 项目 | 字段 |
|------|------|
| 请求签名字段 | 不参与签名 |
| 请求加密字段 | 不参与加密 |
| 响应签名字段 | 不参与签名 |
| 响应加密字段 | 不参与加密 |

### 访问边界

- 只放行本机、私有地址或链路本地地址来源；公网来源即使携带正确令牌也会被拒绝。
- 反向代理转发头中出现公网真实来源时同样拒绝，生产网关仍需禁止公网转发 `/internal/` 路径。
- `ops.config_reload_allowed_ips` 仅用于在内网来源上进一步收窄，留空表示使用默认内网边界。

### 响应示例

```json
{
  "status": true,
  "code": 1000,
  "message": "获取成功",
  "data": {
    "enabled": true,
    "watching": true,
    "configFile": "/app/etc/config.yaml",
    "checkIntervalSeconds": 5,
    "configVersion": "5d41402a",
    "configSummary": "mode=dev app_id=1 sign=0 crypto=0 collector=false",
    "restartRequired": false,
    "restartReason": "",
    "lastStatus": "success",
    "lastMessage": "配置热加载成功",
    "lastTriggerSource": "manual_api",
    "lastFailureCategory": "",
    "lastCheckedAt": "2026-06-03T12:00:00+08:00",
    "lastReloadAt": "2026-06-03T12:00:00+08:00",
    "lastSuccessAt": "2026-06-03T12:00:00+08:00",
    "lastFailureAt": "0001-01-01T00:00:00Z",
    "reloadCount": 1,
    "suppressedFailureCount": 0
  }
}
```

### data 字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| enabled | bool | 是 | false | 配置中是否启用热加载 |
| watching | bool | 是 | false | 当前实例是否正在监听配置文件 |
| configFile | string | 是 |  | 当前监听的配置文件路径 |
| checkIntervalSeconds | int | 是 | 5 | 轮询间隔，单位秒 |
| configVersion | string | 是 |  | 当前生效配置版本 |
| configSummary | string | 是 |  | 当前配置摘要，不包含秘钥明文 |
| restartRequired | bool | 是 | false | 是否存在需重启才完全生效的启动期配置变化 |
| restartReason | string | 否 |  | 需要重启的原因 |
| lastStatus | string | 是 | idle | 最近处理结果：idle、success、failed |
| lastMessage | string | 否 |  | 最近处理说明 |
| lastTriggerSource | string | 否 |  | 最近触发来源 |
| lastFailureCategory | string | 否 |  | 最近失败分类 |
| lastCheckedAt | string | 否 |  | 最近一次检查时间 |
| lastReloadAt | string | 否 |  | 最近一次重载时间 |
| lastSuccessAt | string | 否 |  | 最近一次成功时间 |
| lastFailureAt | string | 否 |  | 最近一次失败时间 |
| reloadCount | int | 是 | 0 | 累计成功重载次数 |
| suppressedFailureCount | int | 是 | 0 | 重复失败日志被限频压制次数 |

## 手动触发配置热加载

`POST /internal/system/config-reload/run`

该接口为内网运维路由，不允许公网访问；请求需要登录态和运维令牌，会立即重新读取 `-f` 绑定的主配置文件。启动期依赖配置变化不会在线重建连接，只会在状态中标记 `restartRequired=true`。

### 请求头

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| Authorization | string | 是 |  | `Bearer <token>` |
| X-Ops-Token | string | 是 |  | 配置热加载运维令牌，需匹配 `ops.config_reload_token` |

### 安全字段

| 项目 | 字段 |
|------|------|
| 请求签名字段 | 不参与签名 |
| 请求加密字段 | 不参与加密 |
| 响应签名字段 | 不参与签名 |
| 响应加密字段 | 不参与加密 |

访问边界同“查询配置热加载状态”。

### 响应说明

响应结构与“查询配置热加载状态”一致。配置文件解析失败时返回业务失败，错误详情进入服务日志。
