# 健康检查接口

## 兼容健康检查

`GET /api/health`

该接口为公开路由，不校验 token，兼容旧探针，响应结构与“存活检查”一致。

### 安全字段

| 项目 | 字段 |
|------|------|
| 请求签名字段 | 不参与签名 |
| 请求加密字段 | 不参与加密 |
| 响应签名字段 | 不参与签名 |
| 响应加密字段 | 不参与加密 |

## 存活检查

`GET /api/live`

该接口为公开路由，不校验 token，不访问外部依赖，供 livenessProbe 使用。

### 安全字段

| 项目 | 字段 |
|------|------|
| 请求签名字段 | 不参与签名 |
| 请求加密字段 | 不参与加密 |
| 响应签名字段 | 不参与签名 |
| 响应加密字段 | 不参与加密 |

### 响应示例

```json
{
  "status": true,
  "code": 200,
  "message": "请求成功",
  "data": {
    "status": "ok",
    "mode": "api",
    "node": "api-1",
    "version": "5d41402a"
  }
}
```

## 就绪检查

`GET /api/ready`

该接口为公开路由，不校验 token，会探测 MySQL 和 Redis，供 readinessProbe 和发布流量切换使用。

`data.dependencies` 由组件生命周期注册表生成；配置命名扩展库时会追加对应的 MySQL 依赖项。

### 安全字段

| 项目 | 字段 |
|------|------|
| 请求签名字段 | 不参与签名 |
| 请求加密字段 | 不参与加密 |
| 响应签名字段 | 不参与签名 |
| 响应加密字段 | 不参与加密 |

### 响应示例

```json
{
  "status": true,
  "code": 200,
  "message": "请求成功",
  "data": {
    "status": "ok",
    "mode": "api",
    "node": "api-1",
    "version": "5d41402a",
    "dependencies": [
      {
        "name": "mysql",
        "status": "ok"
      },
      {
        "name": "redis",
        "status": "ok"
      }
    ]
  }
}
```

### 异常说明

依赖不可用时 HTTP 状态码返回 `503`，业务码返回 `50001`，`data.dependencies` 会带具体依赖、业务码和错误说明。

## 指标抓取

`GET /api/metrics`

该接口为公开路由，不校验 token，供 Prometheus 抓取运行指标使用；生产环境应通过内网、网关白名单或安全组限制访问来源。

### 安全字段

| 项目 | 字段 |
|------|------|
| 请求签名字段 | 不参与签名 |
| 请求加密字段 | 不参与加密 |
| 响应签名字段 | 不参与签名 |
| 响应加密字段 | 不参与加密 |
