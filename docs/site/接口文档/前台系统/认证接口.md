# 认证接口

## 注册

`POST /api/auth/register`

### 请求示例

```json
{
  "username": "demo_user",
  "password": "demo-password",
  "nickname": "Demo",
  "email": "demo@example.com",
  "phone": ""
}
```

### 请求字段说明

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| username | string | 是 |  | 用户名，3-32 位 |
| password | string | 是 |  | 登录密码，最小长度由 `auth.password_min_length` 控制 |
| nickname | string | 否 | username | 昵称 |
| email | string | 否 |  | 邮箱 |
| phone | string | 否 |  | 手机号 |

### 安全字段

| 项目 | 字段 |
|------|------|
| 请求签名字段 | username, password, nickname, email, phone |
| 请求加密字段 | password, email, phone |
| 响应签名字段 | token, expiresAt |
| 响应加密字段 | token, user.email, user.phone |

### 响应示例

```json
{
  "status": true,
  "code": 1100,
  "message": "创建成功",
  "data": {
    "token": "jwt-token",
    "expiresAt": 1790841600,
    "user": {
      "id": 1,
      "username": "demo_user",
      "nickname": "Demo",
      "email": "demo@example.com",
      "phone": "",
      "avatar": "",
      "status": 1,
      "lastLoginAt": "2026-06-02 12:00:00",
      "lastLoginIp": "127.0.0.1",
      "createdAt": "2026-06-02 12:00:00",
      "updatedAt": "2026-06-02 12:00:00"
    }
  }
}
```

注册入口可通过 `auth.register_rate_limit` 配置 Redis 限流，触发后返回业务码 `1003`。注册成功或触发限流时会 best-effort 投递 `auth.security` 脱敏风控事件。

## 登录

`POST /api/auth/login`

### 请求示例

```json
{
  "username": "demo_user",
  "password": "demo-password"
}
```

### 安全字段

| 项目 | 字段 |
|------|------|
| 请求签名字段 | username, password |
| 请求加密字段 | password |
| 响应签名字段 | token, expiresAt |
| 响应加密字段 | token, user.email, user.phone |

### 响应说明

响应结构与注册接口一致。失败时常见业务码：

| 状态码 | 状态含义 |
|------|----------|
| 20001 | 账号或密码错误 |
| 21003 | 账号被禁用 |
| 1003 | 请求过于频繁 |

登录入口可通过 `auth.login_rate_limit` 配置 Redis 限流，默认建议按 IP 和用户名两个维度保护。登录成功、账号密码失败、用户禁用或触发限流时会 best-effort 投递 `auth.security` 脱敏风控事件。

## 刷新令牌

`POST /api/auth/refresh`

### 请求头

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| Authorization | string | 是 |  | `Bearer <token>` |

### 安全字段

| 项目 | 字段 |
|------|------|
| 请求签名字段 | 不参与签名 |
| 请求加密字段 | 不参与加密 |
| 响应签名字段 | token, expiresAt |
| 响应加密字段 | token |

### 响应说明

响应结构与登录接口一致。刷新成功会写入新 `jti` 会话和用户 jti 索引，并删除旧 token 对应的 Redis 会话；旧会话删除失败时本次刷新失败，新会话会回滚。旧 session 在鉴权成功后会 best-effort 懒补 jti 索引。刷新成功后会 best-effort 投递 `auth.security` 脱敏风控事件。

## 退出登录

`POST /api/auth/logout`

### 请求头

| 字段 | 类型 | 是否必须 | 默认值 | 字段解释 |
|------|------|----------|--------|----------|
| Authorization | string | 是 |  | `Bearer <token>` |

### 安全字段

| 项目 | 字段 |
|------|------|
| 请求签名字段 | 不参与签名 |
| 请求加密字段 | 不参与加密 |
| 响应签名字段 | 不参与签名 |
| 响应加密字段 | 不参与加密 |

### 响应示例

```json
{
  "status": true,
  "code": 1,
  "message": "退出登录成功"
}
```

退出成功后当前 token 对应的 Redis 会话和 jti 索引会被删除；后续再次使用该 token 会返回会话失效。退出成功后会 best-effort 投递 `auth.security` 脱敏风控事件。

## 登录态失败事件

需要登录态的接口在 token 缺失、token 过期、Redis session 失效、token 无效、用户不存在或用户禁用时，会 best-effort 投递 `auth.security` 脱敏风控事件；接口响应业务码和 HTTP 状态保持不变。
