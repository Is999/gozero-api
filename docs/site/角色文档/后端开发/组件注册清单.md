# 组件注册清单

本文档记录 `gozero-api` 默认 HTTP 路由模块和轻量运行时扩展入口。新增默认注册项时优先修改 `internal/bootstrap/registrations.go`，避免注册入口散落在 main、bootstrap、handler 或业务逻辑中。

## 统一入口

| 注册类型 | 统一入口 | 说明 |
| --- | --- | --- |
| HTTP 路由 | `internal/bootstrap/registrations.go:defaultRouteModules` | 统一声明内置路由模块顺序，启动时传入 `handler.RegisterHandlers` |
| 路由契约 | `internal/handler/route_contract.go:DefaultRouteContracts` | 统一声明内置路由的 method、path、RouteMeta 和接口文档 |
| 路由元数据 | `internal/handler/route_meta.go:DefaultRouteMetas` | 统一声明路由别名、访问边界和中文说明 |
| 路由安全链路 | `internal/handler/route_security_contract.go:DefaultRouteSecurityContracts` | 统一声明 none/public/auth/internal 安全链路 |
| 路由安全清单 | `internal/handler/route_security_manifest.go:DefaultRouteSecurityManifest` | 汇总前端同步所需的路由、链路和字段级签名加密策略 |
| 前端安全快照 | `docs/site/route_security_manifest.json` | 给前台工程同步 route policy 使用，测试会校验其与后端清单一致 |
| 启动配置校验 | `internal/bootstrap/config_validate.go:validateConfig` | 启动前校验基础配置、生产硬约束和安全秘钥契约 |
| 运行时扩展 | `internal/bootstrap/registrations.go:defaultRuntimeRegistryNames` | 记录组件生命周期、Collector、认证风控事件、默认 Processor、sys_config 缓存和缓存重建锁等轻量扩展入口 |
| 注册清单校验 | `internal/bootstrap/registrations.go:ValidateDefaultRegistrationManifest` | 启动前校验清单完整性、名称唯一性和真实默认注册集合一致性 |

## HTTP 路由模块

| 名称 | 注册文件 | 构造方法 | 实际路由方法 | 说明 |
| --- | --- | --- | --- | --- |
| `health` | `internal/handler/routes.go` | `handler.NewHealthRouteModule` | `registerHealthRoutes` | 存活、就绪和指标路由 |
| `auth` | `internal/handler/routes.go` | `handler.NewAuthRouteModule` | `registerAuthRoutes` | 注册、登录、刷新 token、退出登录 |
| `user` | `internal/handler/routes.go` | `handler.NewUserRouteModule` | `registerUserRoutes` | 前台用户资料路由 |
| `system` | `internal/handler/routes.go` | `handler.NewSystemRouteModule` | `registerSystemRoutes` | 内网配置热加载状态和手动触发路由 |

## HTTP 路由契约

新增内置路由必须同步补 `DefaultRouteContracts()`。契约表会和 go-zero 实际注册路由、接口文档、安全策略做一致性测试，避免路由存在但文档、访问边界或安全策略遗漏。

## 路由访问边界

| 边界 | 说明 |
| --- | --- |
| `public` | 公开入口，不要求前台登录态，例如健康检查、注册和登录 |
| `auth` | 前台登录态入口，必须通过 JWT 与 Redis session 校验 |
| `internal` | 内网运维入口，必须使用 `/internal/` 前缀并限制来源 IP |

## 路由安全链路

| 链路 | 说明 |
| --- | --- |
| `none` | 不经过前台签名、加密或 JWT 链路，仅用于健康检查和指标入口 |
| `public` | 经过签名和加密链路，但不校验 JWT，例如注册和登录 |
| `auth` | 必须校验 JWT 与 Redis session |
| `internal` | 必须校验 JWT、Redis session 和内网 Ops 令牌；同时显式跳过前台签名加密策略，避免运维接口被客户端签名规则误拦 |

## 路由安全清单

`DefaultRouteSecurityManifest()` 会把 `DefaultRouteContracts`、`DefaultRouteSecurityContracts` 和 `RouteSecurityPolicies` 汇总成前端同步清单。新增或修改签名、加密字段后，必须同步 `docs/site/route_security_manifest.json`，前端 route policy 必须按该快照同步，不能自行扩大到整包、大结构体或额外字段。

## 启动配置校验

`validateConfig()` 会在基础设施初始化前执行。`security.secret_key` 未配置时兼容普通请求；配置后必须满足 `app_id`、稳定版本、灰度版本、链路开关和 AES/RSA 材料校验，生产环境启用灰度秘钥时必须配置 `gray_salt`。

## 轻量运行时扩展

| 名称 | 注册文件 | 注册方法 | 说明 |
| --- | --- | --- | --- |
| `component_lifecycle` | `internal/bootstrap/components.go` / `internal/svc/component.go` | `buildDefaultComponentRegistry` / `svc.NewComponentRegistry` | 统一注册核心组件健康探测和关闭入口 |
| `collector_processor` | `internal/collector/manager.go` | `RegisterProcessor` / `RegisterProcessorFunc` | 按 bizType 注册同步或 Redis Stream Processor |
| `auth_security_event` | `internal/logic/auth_event.go` | `logic.AuthCollectorBizType` / `RecordAuthEvent` | 投递脱敏认证风控事件到轻量 Collector |
| `auth_security_processor` | `internal/collector/auth_security.go` | `collector.RegisterDefaultProcessors` / `NewAuthSecurityProcessor` | 默认汇总 `auth.security` 认证风控事件指标，并按 `category` 归并告警分类 |
| `sys_config_cache` | `internal/logic/sys_config.go` | `NewSysConfigLogic` / `GetCachedValue` | 读取 `sys_config` 运行期配置，优先命中 Redis 缓存 |
| `sys_config_key_registry` | `internal/logic/sys_config_key.go` | `NewSysConfigKeyRegistry` / `SysConfigLogic.GetBool` | 声明配置 uuid、类型和默认值，按类型读取运行期配置 |
| `cache_rebuild_lock` | `internal/infra/redsync/lock.go` / `internal/logic/cache_guard.go` | `redsync.WithLock` / `tryRebuildCacheWithLock` | 使用 redsync 保护热点缓存重建 |

## 组件生命周期

核心启动期组件通过 `buildDefaultComponentRegistry()` 注册到 `svc.ComponentRegistry`。`/api/ready` 遍历该清单执行健康探测，服务停止时按注册顺序反向释放资源。

## sys_config 约束

`sys_config` 表固定不加 `api_` 前缀。业务读取运行期配置优先声明 `SysConfigKey` 并使用类型化 getter；底层流程仍是 Redis Hash 命中优先，缺失后通过 redsync 锁保护主库回源，并写入短 TTL 空值占位防穿透。

新增业务运行期配置时优先声明 `SysConfigKey` 并通过 `GetBool`、`GetInt`、`GetString`、`GetFloat`、`GetObject` 或 `GetArray` 读取，避免业务代码散落字符串 uuid 和类型断言。

如果部署库还没有该表，只执行 `internal/database/sys_config_schema.sql.tmpl`；不得新增 `api_sys_config`。

## 扩展规则

1. 新增默认路由模块：先在 `internal/handler/routes.go` 增加 `NewXxxRouteModule`，再加入 `internal/bootstrap/registrations.go:defaultRouteModules` 和 `DefaultRegistrationManifest`。
2. 新增路由必须同步补 `DefaultRouteContracts`、`RouteMeta`、`DefaultRouteSecurityContracts`、接口文档和登录态边界；公开路由要写清公开原因，内网运维路由必须使用 `/internal/` 前缀并限制来源 IP。
3. 新增或修改字段级安全策略后，必须让 `ValidateDefaultRouteSecurityManifest` 通过，并同步 `docs/site/route_security_manifest.json` 与前端同名 route policy。
4. 新增启动期核心组件：在 `buildDefaultComponentRegistry()` 注册 `Name`、`ErrorCode`、`Check` 和 `Close`，避免 ready 检查和停机释放分散在业务逻辑里。
5. 新增 Collector bizType：通过 `Collector.RegisterProcessor` 或 `Collector.RegisterProcessorFunc` 注册处理器，默认仍保持轻量 Collector，不引入后台 taskqueue/outbox；内置认证风控事件使用 `auth.security` 并注册默认指标 Processor，指标标签必须保持低基数。
6. 新增业务运行期配置：先声明 `SysConfigKey`，再使用类型化 getter；只有兼容旧调用链时才直接使用 `GetCachedValue(uuid)`。
7. 新增前台业务表默认使用 `api_` 前缀；复用全局运行期配置时只允许使用无前缀的 `sys_config`。
8. 所有默认注册名称必须唯一；`go test ./internal/bootstrap` 会校验清单和真实注册集合一致。
