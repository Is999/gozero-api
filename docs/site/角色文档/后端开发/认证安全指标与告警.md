# 认证安全指标与告警

本文档用于 `api` 认证、签名和加解密链路的生产观测。指标只做低基数聚合，不写入密码、token、明文用户名、明文 IP 或完整请求体。

## 指标

`api_auth_security_events_total{app_id,action,reason,category}` 由默认 `auth.security` Processor 统计：

| 标签 | 说明 |
| --- | --- |
| `app_id` | 当前站点命名空间，异常值会归并为 `unknown` 或 `other` |
| `action` | 事件动作，例如 `login_failed`、`auth_failed`、`security_failed` |
| `reason` | 细分原因，例如 `signature_failed`、`request_decrypt_failed` |
| `category` | 低基数告警分类，用于 Prometheus 聚合 |

## 分类

| category | 典型原因 | 处理方向 |
| --- | --- | --- |
| `auth` | `invalid_password`、`user_disabled`、`user_not_found` | 关注登录失败基线和账号状态 |
| `token` | `missing_bearer`、`token_expired`、`session_expired`、`token_invalid` | 关注客户端登录态、Redis session 和 JWT 时钟 |
| `rate_limit` | `login_ip_rate_limited`、`login_username_rate_limited`、`register_ip_rate_limited` | 关注暴力尝试和误伤 |
| `security_client` | `security_app_id_invalid`、`signature_failed`、`crypto_disabled`、`request_decrypt_failed` | 关注客户端签名、加密声明、AppID 或秘钥版本传参 |
| `security_config` | `security_key_unavailable` | 优先排查 `security.secret_key`、`sys_config`、Redis 缓存和秘钥材料 |
| `security_payload_limit` | `security_payload_too_large` | 排查大字段、大对象、分页列表进入签名或加密链路 |
| `security_response` | `response_sign_failed`、`response_encrypt_failed` | 优先排查后端响应字段、策略清单和秘钥材料 |
| `session_lifecycle` | `session_created`、`session_rotated`、`current_session_deleted`、`user_sessions_invalidated` | 观察 session 轮换和批量失效行为 |

## 告警建议

以下 PromQL 阈值是示例，生产环境应按实际流量基线调整。

```promql
sum by (app_id, reason) (
  rate(api_auth_security_events_total{action="security_failed"}[5m])
)
```

用于观察签名、加密链路总体失败分布。

```promql
sum by (app_id) (
  rate(api_auth_security_events_total{category="security_config"}[5m])
) > 0
```

`security_config` 出现即应告警，通常代表服务端秘钥、运行期配置或缓存链路异常。

```promql
sum by (app_id) (
  rate(api_auth_security_events_total{category="security_payload_limit"}[5m])
) > 1
```

用于发现大结构体、分页列表或异常请求进入签名/加密解析链路。

```promql
sum by (app_id, reason) (
  rate(api_auth_security_events_total{category="security_client"}[5m])
) > 5
```

用于观察客户端签名错误、解密失败或 AppID 传参异常。上线、灰度和秘钥轮换期间应结合版本头排查。

## 排障边界

- `request_decrypt_failed` 和 `signature_failed` 多数是客户端请求、字段顺序、秘钥版本或 trace 参与签名不一致。
- `security_payload_too_large` 表示请求签名、请求加密、响应签名或响应加密字段超过安全链路上限，不应简单扩大上限；优先减少参与字段。
- `security_key_unavailable` 属于服务端配置问题，先检查 `security.secret_key`、`sys_config` 缓存和 redsync 回源日志。
- `response_sign_failed`、`response_encrypt_failed` 属于后端响应处理问题，先检查 RouteSecurityPolicy、接口文档安全字段和实际响应结构是否一致。
- `/api/metrics` 必须仅允许内网、网关白名单或 Prometheus 来源访问，禁止公网直接暴露。
